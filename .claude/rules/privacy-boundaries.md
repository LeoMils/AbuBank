# Rule: Privacy Boundaries

- Store patterns, not raw conversations.
- Store relationship roles, not unnecessary personal details.
- Keep only what materially improves product behavior.
- Abstract when possible — "close friend" rather than full biographical detail.
- Medical details: never store. Financial details: never store.
- Address: city level only (Kfar Saba), never street.
- Phone numbers: never store in memory files.
- When uncertain if detail is needed: omit. It can be added later if justified.
